# Moz-Gondza

Plataforma educacional moçambicana que permite a estudantes fazer quizzes, acompanhar o progresso e competir numa classificação.

## Run & Operate

- `pnpm --filter @workspace/api-server run dev` — run the API server (port from env)
- `pnpm run typecheck` — full typecheck across all packages
- `pnpm run build` — typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` — regenerate API hooks and Zod schemas from the OpenAPI spec
- `pnpm --filter @workspace/db run push` — push DB schema changes (dev only)
- Required env: `DATABASE_URL` — Postgres connection string, `JWT_SECRET`

## Stack

- pnpm workspaces, Node.js 22, TypeScript 5.9
- API: Express 5
- DB: PostgreSQL + Drizzle ORM
- Validation: Zod (`zod/v4`), `drizzle-zod`
- API codegen: Orval (from OpenAPI spec)
- Build: esbuild (ESM bundle)
- Frontend: React 19 + Vite + Tailwind CSS v4 + wouter

## Where things live

- `artifacts/api-server/` — Express API server (builds to `dist/index.mjs`)
- `artifacts/moz-gondza/` — React/Vite frontend (builds to `dist/public/`)
- `lib/db/` — Drizzle ORM schema + migrations (`src/schema/index.ts` is source of truth)
- `lib/api-client-react/` — Generated TanStack Query hooks + custom fetch
- `lib/api-spec/` — OpenAPI spec (`openapi.yaml` is source of truth for API contract)
- `lib/api-zod/` — Generated Zod schemas from OpenAPI spec

## Architecture decisions

- Contract-first API: OpenAPI spec → codegen → Zod validators (server) + React Query hooks (client)
- JWT auth stored in localStorage; `setAuthTokenGetter` wires the token into every API call
- `setBaseUrl` in `main.tsx` configures cross-origin API calls when `VITE_API_URL` is set (Render)
- esbuild bundles server to a single `index.mjs`; all platform-specific native modules excluded
- CORS allows all origins in development; restricts to `FRONTEND_URL` + `*.onrender.com` in production

## Product

- Autenticação (registo/login com JWT)
- Quizzes por disciplina com resultados imediatos e pontuação
- Classificação (leaderboard) com pódio e perfis públicos linkáveis
- Perfil social público (`/perfil/:userId`) com estatísticas, conquistas e histórico
- Dashboard do utilizador com progresso por disciplina
- Painel de administração (admin@moz-gondza.mz)

## Render Deployment

### First deploy
1. Push the repository to GitHub.
2. In Render dashboard, click **New → Blueprint** and point to the repo — Render reads `render.yaml` automatically.
3. Three resources are created: `moz-gondza-db` (PostgreSQL), `moz-gondza-api` (web service), `moz-gondza-frontend` (static site).
4. After both services are live, copy their URLs:
   - API URL → set `FRONTEND_URL` env var on the **API service** in Render dashboard
   - API URL → set `VITE_API_URL` env var on the **Frontend service**, then trigger a manual redeploy
5. The DB schema is applied automatically in the API build command (`drizzle-kit push`).

### Environment variables to set manually after first deploy

| Service  | Key            | Value                                          |
|----------|----------------|------------------------------------------------|
| API      | `FRONTEND_URL` | `https://moz-gondza-frontend.onrender.com`     |
| Frontend | `VITE_API_URL` | `https://moz-gondza-api.onrender.com`          |

All other variables (`DATABASE_URL`, `JWT_SECRET`, `SESSION_SECRET`) are auto-generated by Render.

### Build commands (managed by render.yaml)
- **API buildCommand**: `corepack enable pnpm && pnpm install --frozen-lockfile && pnpm --filter @workspace/api-server run build && pnpm --filter @workspace/db run push`
- **API startCommand**: `node --enable-source-maps artifacts/api-server/dist/index.mjs`
- **Frontend buildCommand**: `corepack enable pnpm && pnpm install --frozen-lockfile && pnpm --filter @workspace/moz-gondza run build`

## Gotchas

- `pnpm run typecheck` will show pre-existing TS errors in generated files — these don't affect runtime
- esbuild overrides in `pnpm-workspace.yaml` keep only `linux-x64-glibc` binaries; this matches both Replit and Render
- `VITE_API_URL` is a **build-time** variable — changing it requires a frontend redeploy
- The API server listens on `PORT` (default 10000) bound to `0.0.0.0` for Render port-scanning
- Drizzle migrations run as part of the API buildCommand (not startCommand) — schema is applied at deploy time
